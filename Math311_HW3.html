Median:

n<-5
vector<- c(1,seq(from = 5, to = 100, by = 5))
medians<-NULL
median_sims<- NULL
var_sims<-NULL
set.seed(157)
for(j in 1:21){
  for(i in 1:10000){
    dist<-rexp(n,1/vector[j])
    medians[i]<-median(dist)
  }
  median_sims[j]<-mean(medians) #provides expected medians for each set of 5 exp(lambda) random variables
  var_sims[j]<-var(medians)
}

median_biases<-median_sims-vector #these seem to be decreasing at a rate, let's divide by the lambda values
median_sims/vector #all these values seem to be approximate

The above graphs show the MSE for the median estimator (green), the max-min estimator (red) and the middle 3 estimator (blue) to estimate lambda. The x axis is actually lambda values [1,5,10,,400]. Therefore we can see that the median estimator has the smallest MSE, and is therefore the best estimator of the bunch for lambda, followed by max-min, followed by the middle 3. The median is also the easiest to describe analytically.
